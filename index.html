01On approach

From build first to design first.

Essay
—
2 min read

As coding tools and large language models have evolved, my approach has shifted from build first to design first. AI makes it easy to generate code — but it raises the bar for how well that code needs to hold up.

I focus on moving fast without compromising system design. That means clean architecture, readable code, and infrastructure that continues to make sense as systems grow.

My work spans backend engineering, scalable APIs, and MLOps / DevOps — designing systems end-to-end, from model integration to production infrastructure.

I design systems to handle failure as a first-class concern. That includes building self-healing behaviors — fallbacks, retries, and adaptive workflows — so systems remain reliable under real-world conditions, not just ideal ones.

I actively leverage modern AI coding tools in my workflow, but I treat them as accelerators, not substitutes for good engineering judgment.

Today, I specialize in AI engineering: building systems that are not just intelligent, but dependable in production.
